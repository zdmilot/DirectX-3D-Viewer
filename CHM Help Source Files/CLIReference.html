CLI Reference
3D File Tools » CLI Reference

Overview

The command-line interface (cli.js) provides complete feature parity with the browser GUI. Every conversion, export, and labware-generation capability available in the graphical application can be performed non-interactively from a terminal or scripted into a build pipeline. The CLI also exposes additional inspection and batch-validation commands not available in the GUI.

All operations run entirely in Node.js with no npm dependencies — only the built-in fs, path, zlib, readline, and vm modules are used.

Requirements

  • Node.js 14 or later
  • The js/lib/ folder must exist alongside cli.js (contains XFileLoader.js and pako.min.js)

Two Modes of Operation

Non-interactive (one-shot)

Pass a sub-command and its arguments directly on the command line. The tool processes the request, prints output, and exits. This is useful for scripting and automation.

node cli.js <sub-command> [arguments]

Available sub-commands: convert, export-x, generate, generate-default, sbs, --batch, --help.

You can also pass a file path directly to open and inspect it without entering interactive mode:

node cli.js <file.x> [--validate] [--export [out.x]] [--quiet]

Interactive (REPL)

Start the CLI without arguments to enter the interactive prompt (d3d>). Type commands at the prompt, press Enter to execute. Type help for a summary of all available commands, and exit to quit.

node cli.js

File paths with spaces should be wrapped in double quotes at the interactive prompt.

Command Groups

  • Inspection Commands — load, info, sections, tree, meshes, materials, animations, deep
  • Export & Extraction Commands — export, export-textures, export-x
  • Conversion Command — convert
  • Labware Generation Commands — generate, generate-default, sbs
  • Validation Commands — validate, batch, stats

Inspection Commands

These commands load and interrogate .x and .hxx files. Most require a file to be loaded first (via load in interactive mode, or by passing the file path on the command line).

load <file>

Aliases: open

Loads a .x or .hxx file into memory and displays a brief summary. All subsequent inspection commands operate on the loaded file until a different file is loaded.

load C:\labware\myplate.hxx
load "/Users/me/models/test.x"

The command detects whether the file is a raw .x text file, a compressed (MSZIP) .x file, or a .hxx binary container, and decompresses it automatically.

info

Alias: i

Displays detailed metadata about the loaded file:

  • File path and size
  • Format detected (raw .x / compressed .x / .hxx container)
  • For .hxx: embedded section list (model, texture, labels)
  • Frame count, mesh count, material count, animation count parsed from the .x text
  • Vertex and face totals across all meshes

sections

Alias: sec

Lists all sections embedded in the .hxx archive with their names, types, compressed sizes, and original sizes. Has no output for plain .x files.

tree

Alias: t

Prints the frame hierarchy found in the .x file as an indented tree. Each node shows its name, type (Frame / Mesh / other), and the number of direct children.

meshes

Alias: m

Lists every mesh in the loaded file with its name, vertex count, face count, and number of material groups. A totals row shows the aggregate vertex and face counts.

materials

Alias: mat

Lists all Material definitions found in the .x file, including diffuse RGBA, specular RGB, and specular power values.

animations

Aliases: anim, a

Lists all animation sets found in the .x file (name and key-frame count). Most labware .x files contain no animations.

deep

Alias: d

Performs a full three.js-based parse of the loaded file using XFileLoader.js. Provides a more complete analysis than the lightweight text parser, including full bone/skeleton data and a precise per-mesh vertex/face count after index-buffer expansion. The result is printed as a detailed report. This operation may take a few seconds for large files.

Export & Extraction Commands

export [out.x]

Alias: e

Writes the decompressed .x text of the currently loaded file to disk. If out.x is omitted the output file is placed in the same directory as the source with _exported appended to the name.

# Interactive
export
export C:\output\plate_decompressed.x

# One-shot (combined with file load)
node cli.js myplate.hxx --export output\plate.x

This is equivalent to the GUI’s “export decompressed .x” debug option.

export-textures [dir]

Alias: et

Extracts all embedded textures from the loaded .hxx archive and writes them as individual files into dir. If dir is omitted, files are written to a textures/ sub-folder next to the source file. Has no effect on plain .x files that contain no embedded textures.

export-x [input.x] <output> [format]

Alias: ex

Exports a .x or .hxx file to a standard 3D format. This is the CLI equivalent of the GUI’s Export panel.

# One-shot
node cli.js export-x myplate.x output.obj
node cli.js export-x myplate.x output.stl
node cli.js export-x myplate.x output.glb

# Auto-detect format from output extension (no explicit format needed)
node cli.js export-x myplate.hxx exported_plate.glb

# Interactive (uses currently loaded file)
export-x output.obj

The format is inferred automatically from the output file extension. You can also pass it explicitly as the third argument.

FormatOutputNotes
obj (default).obj + companion .mtlMaterials written as Kd RGB values
stlBinary .stlSingle merged mesh, no material info
glbBinary .glb (GLTF 2.0)Per-mesh materials, full PBR metallic/roughness

The input can be given explicitly as the first argument (when it ends in .x or .hxx) or omitted to use the currently loaded file (interactive mode only).

Conversion Command

Converts 3D model files in any supported format into a DirectX .x file. This is the CLI equivalent of the GUI’s Convert to .x panel, including all rotation and mirror transforms.

convert <input> <out.x> [options]

Alias: cv

# One-shot: convert an OBJ to .x
node cli.js convert mypart.obj mypart.x

# With rotation: stand a flat STL upright
node cli.js convert flat_model.stl upright.x --rotate-x 90

# Combine rotation and mirror
node cli.js convert raw.glb corrected.x --rotate-y -90 --mirror-z

# Interactive
convert mypart.obj mypart.x --rotate-x 90 --mirror-x

Input Formats

ExtensionFormat
.objWavefront OBJ (companion .mtl loaded automatically if present)
.stlSTereoLithography (binary or ASCII)
.glbGL Transmission Format binary package
.gltfGL Transmission Format JSON (external .bin loaded automatically)
.xDirectX ASCII mesh (useful for re-exporting with transforms applied)
.hxxHamilton binary container (decompressed before conversion)

Transform Options

FlagDescription
--rotate-x <deg>Rotate all vertices around the X axis by deg degrees
--rotate-y <deg>Rotate all vertices around the Y axis by deg degrees
--rotate-z <deg>Rotate all vertices around the Z axis by deg degrees
--mirror-xMirror all vertices on the X axis (negates X coordinates, flips face winding)
--mirror-yMirror all vertices on the Y axis
--mirror-zMirror all vertices on the Z axis

Multiple rotation and mirror flags can be combined. Rotations are applied first (X, then Y, then Z), followed by mirrors. Face winding order is automatically corrected after mirroring to preserve correct normal direction.

Rotation angles match the GUI rotation buttons: X+90 in the GUI equals --rotate-x 90, Y−90 equals --rotate-y -90, and so on.

Tip: Use export-x to convert in the opposite direction: .x / .hxx → OBJ / STL / GLB.

Labware Generation Commands

These commands generate parametric SBS/ANSI microplate 3D models and export them as .x files. They are the CLI equivalents of the GUI’s Integra XML panel.

generate <xml> [out.x] [--sbs]

Aliases: gen, g

Parses a Hamilton or Integra XML labware definition file, extracts all plate and well geometry parameters, then generates and saves a .x model file.

# One-shot: generate .x from XML
node cli.js generate MyPlate.xml
node cli.js generate MyPlate.xml output/MyPlate.x

# Snap footprint to SBS/ANSI standard dimensions
node cli.js generate MyPlate.xml MyPlate_sbs.x --sbs

# Interactive
generate MyPlate.xml MyPlate.x --sbs

If out.x is omitted, the output file is saved in the same directory as the XML file with the same base name and a .x extension.

The --sbs flag snaps the plate footprint length (127.76 mm), width (85.48 mm), and well spacing to ANSI/SLAS standard values, overriding values in the XML. Useful when the XML contains slightly non-standard dimensions.

XML Format

The parser handles the Integra/Hamilton XML labware format. Key fields extracted:

XML Element / AttributeCLI parameter
NameLabware name
NoOfWellsX / NoOfWellsYColumns / rows
WellDistX / WellDistYColumn/row spacing (div 100 → mm)
FirstHolePositionX / FirstHolePositionY (or FirstHolePositionText)A1 X/Y offset
WellDiameter (or TopInnerDiameter)Well top diameter
WellDepthWell depth
HeightTotal plate height
Length / WidthFootprint dimensions

All dimension values in the XML use DIV=100 encoding (integer tenths of micrometers) unless otherwise noted; the parser divides by 100 automatically to convert to millimeters.

generate-default [out.x] [options]

Aliases: gend, gd

Generates a .x model using built-in defaults and command-line parameters — no XML file required. This is the fastest way to create a standard labware model from scratch.

# Standard 96-well plate with SBS footprint
node cli.js generate-default MyPlate96.x --rows 8 --cols 12 --sbs

# 384-well plate, round-bottom wells
node cli.js generate-default MyPlate384.x --rows 16 --cols 24 --sbs \
    --well-size 3.3 --well-depth 11.5 --bottom-shape circle

# 24-well plate, v-bottom, labelled
node cli.js generate-default "24 Well Plate.x" \
    --rows 4 --cols 6 --name "24-Well Plate" \
    --well-shape circle --bottom-shape vshape \
    --well-size 15.5 --well-depth 17.4

# Interactive
generate-default output.x --rows 8 --cols 12 --sbs

Generated Geometry

The geometry engine generates the following mesh components:

ComponentDescription
FlangeUnderside ledge that sits in the deck cutout
Outer wallsFour vertical side walls of the plate body
SlabThe solid base of the plate between well bottoms and the underside
Top surfaceFlat top face with circular/rectangular holes cut around each well
Well wallsTapered cylindrical or rectangular well walls from opening to floor
Well bottomFlat disc, hemispherical cap, or conical (V-bottom) floor per well

Circular cross-sections are approximated with 16-sided polygons, matching the GUI generator exactly. A typical 96-well plate produces ~200 meshes (~8,000 vertices, ~5,000 faces).

Validation Commands

validate [file]

Alias: v

Attempts a full parse of the specified file (or the currently loaded file) using the XFileLoader pipeline. Reports whether the parse succeeded and lists any errors or warnings encountered. Exit code is 0 on success, 1 on parse failure.

node cli.js myplate.hxx --validate
# Or in interactive mode after loading:
validate

batch <dir> [--recursive] [--no-recursive]

Alias: b

Recursively (default) or non-recursively (--no-recursive) validates every .x and .hxx file found in dir. Prints a pass/fail status for each file and a summary at the end. Combine with stats to see aggregate totals.

node cli.js --batch C:\Hamilton\Labware
node cli.js --batch ./models --no-recursive

stats

Alias: s

Prints aggregate statistics from the last batch run: total files, pass/fail counts, total vertices and faces parsed, elapsed time.

Workflow Examples

Example 1: Export a .hxx Model as GLB for Use in Blender

node cli.js export-x MyLabware.hxx MyLabware.glb

Decompresses the .hxx container, loads the embedded .x mesh, and writes a GLTF 2.0 binary (.glb) file readable by Blender, Godot, and other 3D applications.

Example 2: Convert an STL from a 3D Printer to .x

node cli.js convert printed_part.stl labware.x --rotate-x -90

Loads the STL, rotates −90° around X (common correction when STL files are oriented with Z-up instead of Y-up), and writes a DirectX .x file.

Example 3: Generate a Standard 96-Well Plate .x from Scratch

node cli.js generate-default MyPlate96.x --rows 8 --cols 12 --sbs --name "Generic 96-Well Plate"

Creates a complete 3D plate model with SBS standard footprint, standard 9.0 mm well spacing, and the specified name. No XML or source file required.

Example 4: Generate a 384-Well Plate with Round-Bottom Wells

node cli.js generate-default MyPlate384.x \
    --rows 16 --cols 24 --sbs \
    --well-size 3.3 --well-depth 11.5 \
    --bottom-shape circle \
    --name "384-Well Round Bottom"

Example 5: Round-Trip Validation — Convert a GLB and Verify

node cli.js convert scene.glb output.x
node cli.js output.x --validate --quiet

Converts the GLB file to .x, then validates that the output can be fully parsed by the XFileLoader pipeline. Exit code 0 means success.

Example 6: Batch-Validate an Entire Labware Library

node cli.js --batch C:\Hamilton\Library\Labware
# Then in interactive mode to see aggregate stats:
node cli.js
d3d> batch C:\Hamilton\Library\Labware
d3d> stats

Example 7: Generate .x from Integra XML

node cli.js generate PlateDefinition.xml PlateDefinition.x --sbs

Parses the XML, applies SBS standard footprint overrides, and saves the generated model to PlateDefinition.x.
