Overview

3D File Tools supports reading and writing several 3D file formats. The table below summarizes which formats are supported in each direction and which tools use them.

DirectX .x Format

The .x format is a DirectX ASCII text mesh file used by Hamilton VENUS to define 3D geometry for labware visualization. It stores Frame hierarchies, vertex positions, face indices, normals, and material properties.

See DirectX .x File Format for technical details.

Hamilton .hxx Format

The .hxx format is a Hamilton-proprietary binary container that wraps a gzip-compressed .x file (plus optional PNG texture sections) in a custom binary envelope called Hamilton3dData.

See Hamilton .hxx Format for technical details.

Wavefront OBJ

OBJ is a plain-text format that stores vertex positions (v), texture coordinates (vt), normals (vn), and face definitions (f). An optional companion .mtl file defines material properties including diffuse color, specular, opacity, and texture maps.

When loading OBJ files in the Converter, if a .mtl file with the same base name exists in the same folder, materials are automatically applied. You can also multi-select both files in the file picker.

STereoLithography (STL)

STL stores a list of triangles with surface normal vectors. Both binary and ASCII STL are supported for reading. The exporter writes binary STL. STL has no material or color support — all geometry is exported with a default gray material.

GL Transmission Format (GLB/GLTF)

GLTF 2.0 is a modern format that supports PBR materials, textures, skeletal animation, and scene hierarchies. GLB is the binary-packaged variant that bundles all assets in a single file. GLTF (with separate .bin and texture files) is supported for reading only when all referenced files are available.

XML Labware Definition

The Integra Generator can import Hamilton or Integra XML labware definition files to pre-populate the parameter form. The parser extracts plate footprint dimensions, well count, well geometry, well spacing, and A1 position. The XML is not saved or exported — it is used only as an input for generating the parametric 3D model.

PNG / JPEG / SVG

These image formats are used only by the Screenshot feature. See Screenshot.
