Hamilton .hxx Format
3D File Tools » File Formats » Hamilton .hxx Format

Overview

The .hxx format is a Hamilton-proprietary binary container called Hamilton3dData. It wraps one or more named sections, each containing gzip-compressed data. The primary section (__Main3dData__) holds a standard DirectX ASCII .x file. Additional sections may hold PNG texture images.

The 3D Viewer reads .hxx files transparently — from the user's perspective, they behave identically to .x files. The HXX Loader decompresses the .x data on-the-fly and passes it directly to the XFileLoader.

Section Table Entry (12 bytes each)

Byte OffsetSizeFieldDescription
04 bytesData OffsetBig-endian uint32: byte offset from file start to the section’s data block
44 bytesName LengthBig-endian uint32: length of the section name in bytes
84 bytesDecompressed SizeBig-endian uint32: expected size of the decompressed data

Section Data Block

Each section data block consists of:

  1. Section Name — ASCII string, nameLength bytes, no null terminator
  2. Compressed Data — gzip-compressed payload. Compressed size is implied: from (offset + nameLength) to the next section’s offset (or end of file for the last section)

Section Names

Section NameContent
__Main3dData__The main gzip-compressed DirectX .x ASCII text file
Other names (e.g. texture paths)gzip-compressed PNG texture data (not currently rendered by the viewer)

Gzip Decompression

The HXX Loader uses pako (a pure-JavaScript zlib/gzip implementation) to decompress section data. The loader:

  1. Parses the gzip header to find the DEFLATE payload start
  2. Locates the ISIZE trailer field (last 4 bytes of the gzip stream) to find the exact end of the compressed stream
  3. Calls pako.inflate() on the raw DEFLATE data with raw: true
  4. Decodes the resulting bytes as UTF-8 to recover the .x text

Detection

The HXX Loader checks for the Hamilton3dData magic header at offset 0 before attempting to parse. Files that do not begin with this magic string are rejected. This check is exposed as HXXLoader.isHXX(arrayBuffer).

Writing .hxx Files

The Export as .HXX option in the 3D Viewer creates a valid .hxx container from the current .x model data. The .x text is gzip-compressed and wrapped in a single-section Hamilton3dData envelope with the __Main3dData__ section name.
