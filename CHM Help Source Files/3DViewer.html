Overview

The 3D Viewer is the primary tool for loading, inspecting, and interacting with DirectX .x and Hamilton .hxx 3D model files. It is the default panel displayed when the application starts.

Opening a File

There are two ways to load a file into the viewer:

  • Open button — Click Open in the bottom controls bar. A file browser dialog opens. Select a .x or .hxx file.
  • Drag and drop — Drag a .x or .hxx file from your file manager and drop it onto the viewport. A drop zone indicator appears while dragging.

After loading, the camera automatically fits to the model bounding box and the file name is displayed in the bottom bar.

Floating Toolbar

The floating toolbar appears on the left side of the viewport. It can be collapsed using the chevron toggle button, and repositioned by dragging the grip handle at the bottom of the toolbar.

Transform Flyout

Click the Rotate / Mirror button (sync icon) in the floating toolbar to open the Transform flyout panel. The flyout provides:

Rotate ±90°

Six buttons to rotate the model 90° in either direction around each axis:

ButtonAction
X +90°Rotate 90° around the X axis (positive direction)
X −90°Rotate 90° around the X axis (negative direction)
Y +90°Rotate 90° around the Y axis (positive direction)
Y −90°Rotate 90° around the Y axis (negative direction)
Z +90°Rotate 90° around the Z axis (positive direction)
Z −90°Rotate 90° around the Z axis (negative direction)

Rotations are applied to the underlying .x file geometry data. You can save the transformed model back to a .x file using Save.

Wireframe Mode

Click the Wireframe button (polygon icon) in the floating toolbar to toggle wireframe rendering. In wireframe mode, solid surfaces are replaced by edge lines, making it easier to inspect mesh topology and face count.

Perspective and Orthographic Projection

Click the Perspective/Orthographic button (cube icon) in the toolbar to toggle between:

  • Perspective (default) — Objects farther away appear smaller, giving a natural 3D feel
  • Orthographic — All geometry is the same scale regardless of distance; useful for precise measurement and alignment

Grid

A reference grid is shown at the base of the model by default. Toggle it on/off with the Grid button (border-all icon) in the toolbar. The grid adapts to the model scale and updates when a new file is loaded.

Axis Gizmo

The axis gizmo at the top of the floating toolbar shows the current orientation of the X (red), Y (green), and Z (blue) world axes relative to the camera. It updates live as you orbit.

Debug Panel

The debug panel is hidden by default. To reveal it:

  1. Click the About button (ⓘ) to open the About dialog
  2. Click the teapot logo twice (so it rotates 180° and appears upside-down)
  3. Close the About dialog

The debug panel displays mesh statistics for the loaded model: mesh count, animation count, per-mesh vertex count and first vertex coordinates, NaN vertex warnings, bounding box, size, max dimension, and scene child count. Click the teapot one more time (270°) or once more (back to 0°) to dismiss.

Axis Gizmo

The compact axis gizmo at the top of the floating toolbar shows an XYZ orientation indicator that tracks the camera. X = red, Y = green, Z = blue.

Technical Details

The 3D Viewer is built on Three.js r152 with the following rendering enhancements:

  • Logarithmic depth buffer — Eliminates z-fighting on large models with wide depth ranges
  • Ear-clipping triangulation — Correctly renders concave polygons found in .x files
  • Double-sided materials — All faces are visible from both sides regardless of winding order
  • Overlapping mesh separation — Automatically detects and nudges coplanar meshes apart
  • Per-mesh polygon offset — Depth bias prevents z-fighting between overlapping sub-meshes

See XFileLoader Rendering Pipeline for full technical details.
