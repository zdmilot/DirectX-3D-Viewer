DirectX .x File Format
3D File Tools » File Formats » DirectX .x File Format

Overview

The DirectX .x format is an ASCII text mesh format used by DirectX 8 and earlier versions. Hamilton VENUS uses it extensively for 3D labware visualization. A .x file defines a hierarchy of named Frames, each of which can have a transformation matrix and one or more Meshes.

File Structure

A minimal .x file has the following structure:

xof 0303txt 0032

Frame RootFrame {
    FrameTransformMatrix {
        1.000000, 0.000000, 0.000000, 0.000000,
        0.000000, 1.000000, 0.000000, 0.000000,
        0.000000, 0.000000, 1.000000, 0.000000,
        0.000000, 0.000000, 0.000000, 1.000000;;
    }

    Mesh MeshName {
        <vertex count>;
        <x>;<y>;<z>;,
        ...

        <face count>;
        <vertex count per face>; <i0>, <i1>, <i2>;,
        ...

        MeshNormals {
            <normal count>;
            <nx>;<ny>;<nz>;,
            ...

            <face normal index count>;
            <count>; <ni0>, <ni1>, <ni2>;,
            ...
        }

        MeshMaterialList {
            <material count>;
            <face count>;
            <face material index>, ...;

            Material {
                <r>;<g>;<b>;<a>;;
                <power>;
                <specR>;<specG>;<specB>;;
                <emissR>;<emissG>;<emissB>;;
            }
        }
    }
}

Key Nodes

NodeDescription
xof 0303txt 0032Magic header. 0303 = version 3.3, txt = ASCII text format, 0032 = 32-bit float precision.
FrameNamed coordinate frame. Can contain a FrameTransformMatrix and child Frame or Mesh nodes. Frames are nested to build a scene hierarchy.
FrameTransformMatrixA 4×4 row-major transformation matrix (16 floats) defining the local-to-parent transform for the frame. Written in row-major order: rows 0–3, columns 0–3.
MeshThe main geometry container. Contains a vertex list, a face index list, and child MeshNormals and MeshMaterialList sub-nodes.
MeshNormalsPer-vertex normal vectors and per-face normal index lists. One normal entry per vertex of each face (may differ from the vertex index list).
MeshMaterialListDefines how many materials the mesh uses, one material index per face, and the Material sub-nodes.
MaterialRGBA face color (diffuse), specular power, specular RGB, and emissive RGB.

Face Format

Faces can have any number of vertices (n-gons). The face line format is:

<vertex count per face>; <v0>, <v1>, ..., <vN-1>;,

Triangles use 3;, quads use 4;, and pentagons and higher use their respective count. 3D File Tools uses ear-clipping triangulation to correctly decompose concave n-gon faces during rendering. See XFileLoader Rendering Pipeline for details.

Coordinate System

DirectX .x files use a left-handed coordinate system with Y-up orientation. Three.js uses a right-handed Y-up system. The XFileLoader handles any necessary axis adjustments during parsing.

Character Encoding

.x files are plain ASCII text. Numeric values use float notation with up to 6 decimal places. Block delimiters are { and }. List items are separated by commas; the last item in a list is followed by a semicolon.

Limitations

  • Only the ASCII text variant (txt) is supported. Binary .x files are not supported.
  • Animation data (AnimationSet, Animation) is parsed but not currently played back in the viewer
  • Texture references in TextureFilename nodes are parsed but texture images are not loaded from disk
