Common Issues

The page opens but the 3D viewer is blank / black canvas

Cause: WebGL is not available or has been disabled in the browser.

Solution:

  • Try a different browser. Chrome and Edge have the best WebGL support.
  • If using Chrome, navigate to chrome://flags/#disable-webgl and ensure WebGL is not disabled.
  • On some systems, hardware acceleration is disabled. Enable it in browser settings under Advanced → System → Use hardware acceleration
  • On some systems, hardware acceleration is disabled. Enable it in browser settings h3>Model loads but looks completely black or invisible

    Cause: The model has no materials, or all materials have black (0, 0, 0) diffuse color.

    Solution:

    • Toggle Wireframe mode in the toolbar to confirm the geometry is present. If wireframe shows correctly, the issue is material color.
    • Some .x files export with black materials by default. This is a property of the source file and cannot be corrected by the viewer. Use a 3D modeling tool to reassign material colors before re-exporting.

    Model loads but appears inside-out or has missing faces

    Cause: Face winding order is inconsistent. This is common in models exported from some CAD tools.

    Solution:

    • All materials are set to THREE.DoubleSide, so inside-out faces should be visible. If faces are still missing, the geometry may have zero-area degenerate triangles.
    • Toggle Wireframe to inspect the mesh topology.
    • Re-export the model from your CAD tool with consistent face normals and winding order.

    Z-fighting / flickering surfaces on a model with multiple parts

    Cause: Coplanar surfaces from two different meshes occupy the same depth in the depth buffer.

    Solution:

    • The viewer automatically detects overlapping meshes and applies nudges and polygon offsets. In most cases this resolves the issue automatically.
    • If z-fighting persists, zoom in on the affected area to increase depth-buffer precision. The logarithmic depth buffer provides much greater precision than a standard depth buffer.
    • For severe cases (e.g. two perfectly coincident surfaces), the overlap separation algorithm may not be able to resolve the conflict without modifying the source geometry.

    Concave polygon faces render with spurious triangles crossing the interior

    Cause: This was a known issue in earlier versions caused by fan triangulation. It is fixed in the current version using ear-clipping triangulation.

    Solution:

    • Ensure you are using the current version of js/lib/XFileLoader.js. The version date appears in the script tag in index.html (e.g. XFileLoader.js?v=20260304a).
    • If the issue persists on a specific model, the polygon may be degenerate (self-intersecting). The ear-clipping algorithm falls back to fan triangulation for degenerate geometry. Pre-triangulate the mesh in your modeling tool.

    Dropped file does not load / drag-and-drop does not work

    Cause: Some browsers restrict drag-and-drop from certain locations (e.g. the Windows desktop). Security policies may also block file:// access.

    Solution:

    • Use the Open button instead of drag-and-drop.
    • Serve the application over HTTP instead of opening it as a file:// URL. Use the included nocache_server.py: 
      python3 nocache_server.py
      Then open http://localhost:8000 in your browser.

    OBJ file loads without colors (all grey)

    Cause: The .mtl material file was not included. OBJ files reference materials by name; without the MTL file, materials default to grey.

    Solution:

    • When using the file picker, select both the .obj and .mtl files at once using multi-select (hold Shift or Ctrl while clicking).
    • Alternatively, ensure the .mtl file is in the same directory as the .obj file with the same base name.

    Export .x button is disabled in the Converter

    Cause: No model has been loaded yet. The export button is disabled until a valid 3D file is loaded.

    Solution: Click Open and select a supported file (OBJ, STL, GLB, or .x) before attempting to export.

    The Integra XML preview appears empty after clicking Generate

    Cause: Invalid parameter values (e.g. zero rows or columns, zero well depth, non-numeric footprint dimensions).

    Solution:

    • Verify that all required fields contain valid positive numbers.
    • Rows and Columns must both be at least 1.
    • Footprint dimensions must be positive.
    • Click Apply SBS Defaults to reset to known-good values, then make your adjustments.

    Ruler measurement does not appear / ruler tool unresponsive

    Cause: Ruler requires a generated model to be present. It measures 3D surface points so geometry must exist.

    Solution:

    • Click Generate 3D Model first, then activate the ruler.
    • If the ruler was accidentally left active while switching tools, press Esc to cancel it.

    HXX file fails to load with a parsing error

    Cause: The file does not begin with the Hamilton3dData magic header, or the gzip-compressed data is corrupt.

    Solution:

    • Confirm the file was exported by Hamilton VENUS or 3D File Tools. Files renamed from another format will not have the correct magic header.
    • Try re-exporting the file using the Export as .HXX option in the 3D Viewer.
    • Open the browser developer console (F12) to see the specific parsing error message.

    Application opens with a white screen / JavaScript error

    Cause: A required JavaScript file failed to load. This usually happens when the application is opened directly as a file:// URL and the browser blocks local script loading.

    Solution:

    • Serve the application over HTTP using the included server: 
      python3 nocache_server.py
      Then navigate to http://localhost:8000.
    • Open the browser developer console (F12 → Console tab) to find the specific error.
