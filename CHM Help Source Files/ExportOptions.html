Export Options
3D File Tools » Export Options

Overview

3D File Tools can export models in several formats. Export is available from the Export dropdown button in the bottom controls bar of the 3D Viewer and Plate Placer, and from the Export .x button in the Converter. The Integra XML has a dedicated Save as .x File button.

Export Formats

DirectX .x (3D Viewer — Save; Converter — Export .x; Integra XML — Save as .x)

Saves the model as a DirectX ASCII text mesh (.x). This is the native format read by Hamilton VENUS. For the 3D Viewer, the saved file includes any rotation or mirror transforms applied via the Transform flyout. For the Converter, the cumulative rotation state at the time of export is baked in. For the Integra XML, the parametric geometry is serialized.

See DirectX .x File Format for format details.

OBJ — Export as .OBJ

Exports to Wavefront OBJ format. The geometry is written as a .obj text file. If the model has materials with non-default colors, an accompanying .mtl file is also generated. OBJ is widely supported by 3D modeling tools including Blender, Maya, and SolidWorks.

  • Vertices are written as v x y z lines
  • Normals are written as vn nx ny nz lines (if present)
  • Faces are written as f v1//n1 v2//n2 v3//n3 triangles
  • Material groups use usemtl directives

STL — Export as .STL

Exports to STereoLithography binary STL format. STL contains only triangle geometry with no material or color information. It is widely used for 3D printing and CAD import. The exported file is a binary STL with an 80-byte header and one 50-byte record per triangle.

GLB — Export as .GLB

Exports to GL Transmission Format binary container (.glb). GLB is a compact binary format that packages geometry, materials, textures, and scene hierarchy in a single file. It is supported by web viewers, Blender, Windows 3D Viewer, and many game engines.

  • Uses the Three.js GLTFExporter
  • Includes material colors and basic PBR material properties
  • Exported as GLTF 2.0 specification

HXX — Export as .HXX (3D Viewer only)

Exports to Hamilton .hxx binary container format. The HXX container wraps a gzip-compressed .x file in the Hamilton3dData binary envelope. The exported file is readable by Hamilton VENUS and any tool that supports the HXX format.

See Hamilton .hxx Format for format details.

File Naming

Exported files are named based on the currently loaded file:

  • 3D Viewer Save (.x): Uses the loaded file name (e.g. mypart.x)
  • Converter Export (.x): Uses the original input file base name (e.g. importing mypart.stl exports mypart.x)
  • Viewer Export (OBJ/STL/GLB/HXX): Uses the loaded file base name with the new extension
  • Integra XML Save (.x): Uses the Name field from the parameter form
