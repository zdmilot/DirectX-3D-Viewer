Introduction

This guide walks you through opening the application and performing the most common operations: viewing a 3D model, converting a file to .x format, and generating labware geometry.

Opening the Application

Open index.html in any modern web browser. The application starts with a brief splash screen showing the teapot logo, then transitions to the main interface.

No installation is required. The application runs entirely in the browser with all dependencies bundled in the js/lib/ folder.

Application Structure

The left sidebar contains four navigation buttons:

ButtonTool
3D ViewerLoad and inspect .x / .hxx files
Convert to .xImport OBJ/STL/GLB and export as DirectX .x
Plate PlacerOverlay a model on a 96-well plate for offset calculation
Integra GenParametric SBS labware generator

Click the hamburger icon (☰) in the top-left to expand the sidebar and show labels next to each icon.

Opening a 3D File in the Viewer

  1. Click the 3D Viewer button in the sidebar (it is selected by default on launch)
  2. Click Open in the bottom controls bar, or drag and drop a .x or .hxx file onto the viewport
  3. The model loads and the camera automatically fits to the model bounds
  4. Orbit with the left mouse button, pan with the right mouse button (or middle mouse), zoom with the scroll wheel
PLACE IMAGE OF VIEWER WITH MODEL LOADED HERE

Supported Input Formats for the Viewer

  • .x — DirectX ASCII mesh format (used by Hamilton VENUS)
  • .hxx — Hamilton 3D Data binary container (wraps a gzip-compressed .x file)

Converting a File to .x Format

  1. Click Convert to .x in the sidebar
  2. Click Open and select an OBJ, STL, or GLB file, or drag and drop it into the viewport
  3. The model appears in the main viewport. Six orthographic views (Top, Front, Right, Bottom, Back, Left) update automatically
  4. Use the rotation buttons (X+90, Y-90, etc.) to orient the model correctly
  5. Click Export .x to save the model as a DirectX .x file

See Convert to .x for full details.

Using the Plate Placer

  1. First load a model in the 3D Viewer (the Plate Placer will auto-load the same model)
  2. Click Plate Placer in the sidebar
  3. The tool shows your model alongside the standard 96-well plate reference model
  4. Adjust the Model Position and Plate Position fields to align the objects
  5. Read the Calculated Offset (ΔX, ΔY, ΔZ) values for use in your labware definition

See Plate Placer for full details.

Generating Labware Geometry

  1. Click Integra Gen in the sidebar
  2. Either import an XML labware definition file, or fill in the parameter form manually
  3. Configure: Name, footprint dimensions, well count, well geometry, A1 position
  4. Click Generate 3D Model to create the 3D preview
  5. Click Save as .x File to export, or Load into Viewer to inspect it in the 3D Viewer

See Integra Generator for full details.

Using the CLI

All GUI operations can also be performed from the command line using cli.js. Run node cli.js --help to see all available commands, or start the interactive prompt with node cli.js.

Common taskCLI command
Export a .x file as OBJnode cli.js export-x model.x model.obj
Convert an STL to .xnode cli.js convert part.stl part.x --rotate-x -90
Generate a 96-well plate .xnode cli.js generate-default plate96.x --rows 8 --cols 12 --sbs
Generate from XML labware definitionnode cli.js generate MyPlate.xml MyPlate.x
Batch-validate a labware directorynode cli.js --batch C:\Hamilton\Labware

See CLI Reference for complete documentation of all commands, flags, and workflow examples.

Theme Toggle

Click the moon icon (☾) in the top-right navigation bar to switch between light and dark rendering modes. All viewports adapt simultaneously.

Introduction

This guide walks you through the initial setup of Venus Library Manager and the most common workflows you will perform when managing Hamilton VENUS libraries.

Installation

Venus Library Manager is distributed as part of your Hamilton VENUS installation or as a standalone download. No separate installer is required - the application runs directly from its directory.

Directory Structure

The application directory contains:

ItemDescription
package.jsonNW.js application manifest and Node.js dependency declarations
cli.jsCommand-line interface entry point (all CLI commands)
cli-schema.jsonJSON Schema for package specification files
cli-spec-example.jsonExample package specification file
html/GUI application HTML, CSS, and JavaScript files
db/Application database: settings, system library definitions, baseline hashes
docs/Technical reference documentation

First-Time Setup

User Data Directory

Venus Library Manager stores its working data (installed library records, group assignments, navigation tree) in a user data directory. By default, this is located at:

C:\Program Files (x86)\HAMILTON\Library\VenusLibraryManager

This directory is created automatically on first run and seeded with empty JSON files for each data collection (installed_libs.json, groups.json, tree.json, links.json).

You can change the user data directory location in the application settings or by passing --db-path on the CLI.

Package Store

Every imported .hxlibpkg file is automatically cached in the package store for future repair and version rollback. The default location is:

C:\Program Files (x86)\HAMILTON\Library\LibraryPackages

Packages are organized into subdirectories by library name, with timestamped filenames for version tracking. See Version Rollback for details.

Basic Workflows

Import a Library Package

  1. Obtain a .hxlibpkg file from a library author or export
  2. In the GUI: Use the Import tab and select the file, or drag-and-drop
  3. Review the pre-install preview (metadata, file lists, install paths)
  4. Confirm the import
  5. Library files are extracted to HAMILTON\Library\<LibraryName>
  6. Demo method files are extracted to HAMILTON\Methods\Library Demo Methods\<LibraryName>

From the CLI:

node cli.js import-lib --file MyLibrary.hxlibpkg

Export a Library

  1. In the GUI: Open the library detail modal and click Export Library
  2. Choose Export Single Library for a .hxlibpkg, or Export with All Dependencies for a .hxlibarch that includes all required dependency libraries
  3. Choose a save location - the current on-disk files are packaged into the selected format

From the CLI:

node cli.js export-lib --name "MyLibrary" --output MyLibrary.hxlibpkg

Create a New Package

  1. In the GUI: Use the Library Packager to select source files and fill in metadata
  2. Or create a JSON spec file and use the CLI:
node cli.js create-package --spec MyLib.spec.json --output MyLib.hxlibpkg

See Creating Packages and Package Spec JSON Schema for full details.
